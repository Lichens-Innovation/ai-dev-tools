# Project: helper-server

## Stack & Integrations

- **Framework**: TanStack Start (React, SSR-first, Vite-based)
- **Router**: TanStack Router (file-based, `src/routes/`)
- **UI**: shadcn/ui + Tailwind CSS v4 (`components.json` at root)
- **State**: TanStack Store (`src/store/sidebar-store.ts`, `src/store/search-store.ts`, `src/store/stats-store.ts`)
- **Linting**: ESLint (`eslint.config.js`) + Prettier (`prettier.config.js`)
- **Package manager**: yarn

## Key Architecture Decisions

- **Server routes pattern**: API endpoints use `createServerFn` from `@tanstack/react-start` for server-side logic; never expose secrets client-side
- **Server functions**: Use `createServerFn` (from `@tanstack/react-start`) for server-side logic; never expose secrets client-side
- **shadcn components**: Add via `npx shadcn@latest add <component>` — components land in `src/components/ui/`
- **Route tree**: Auto-generated at `src/routeTree.gen.ts` — never edit manually

## Project Structure

```
src/
  routes/
    __root.tsx          # Root layout (HeadContent, Scripts, Outlet)
    index.tsx           # Home page — Command Center dashboard
    docs/
      $slug.tsx         # Documentation page
  components/
    Header.tsx / Footer.tsx / ThemeToggle.tsx / CopyableText.tsx / Sidebar.tsx
    tabs/
      CommandCenter.tsx    # Installed plugins & commands tab
      StatsTab.tsx         # Usage stats tab
      MarketplaceTab.tsx   # Project marketplace & rules tab
      CuratedTab.tsx       # Curated external plugins tab
  store/
    search-store.ts     # Search state store
    sidebar-store.ts    # Sidebar toggle state
    stats-store.ts      # Usage stats cache state
  utils/
    helpers.ts          # Shared parsing utilities and path constants
    stats.ts            # Usage stats server functions
    plugins.ts          # Installed & project marketplace plugins
    marketplace.ts      # Curated external marketplace plugins
    rules.ts            # Project rules server functions
    commands.ts         # Claude Code command parsing
    docs.ts             # Docs content server functions
  router.tsx            # createRouter() factory
  styles.css            # Tailwind entry
```

## Development

```bash
yarn dev          # Start dev server (default: http://localhost:3008)
yarn build        # Production build
yarn start        # Serve production build
yarn lint         # ESLint
```

## Known Gotchas

- `--tailwind` flag is deprecated — Tailwind v4 always included; config lives in `styles.css` not `tailwind.config.*`
- Route tree (`src/routeTree.gen.ts`) is auto-generated by the Vite plugin — commit it, never hand-edit
- shadcn components require manual install per component (`npx shadcn@latest add button`)
