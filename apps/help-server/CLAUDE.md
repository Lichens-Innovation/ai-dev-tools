# help-server

A dashboard app that surfaces Claude Code tooling — installed plugins, project marketplace, curated plugins, rules, CLI commands, and documentation — in a single browser UI. Launched via the `ai-tools-manager:help-server` skill.

## Stack

- **Framework**: TanStack Start (React 19, SSR-first, Vite-based)
- **Router**: TanStack Router (file-based, `src/routes/`)
- **UI**: shadcn/ui + Tailwind CSS v4 (`components.json` at root; config lives in `styles.css`, not `tailwind.config.*`)
- **State**: TanStack Store (`src/store/sidebar-store.ts`, `src/store/search-store.ts`, `src/store/stats-store.ts`)
- **Linting/formatting**: ESLint (`eslint.config.js`) + Prettier (`prettier.config.js`)
- **Package manager**: yarn (workspace `helper-server`)
- **Shared package**: `@repo/claude-fs` for all `~/.claude/` reads

## Architecture

The app runs in Docker and reads from two sources:

1. **Host `~/.claude/`** — mounted read-only at `/root/.claude` inside the container. Accessed via `@repo/claude-fs` to get installed plugins and cached marketplace data.
2. **Project repo root** — mounted at `/app` (read-only subdirs for `plugins/`, `rules/`, `docs/`, `.claude-plugin/`). Accessed directly via path constants in `src/utils/helpers.ts`.

All data fetching happens in server functions (`createServerFn` from `@tanstack/react-start`). Never expose secrets client-side.

## Routes

| Route | Purpose |
|---|---|
| `/` | Main dashboard — tabbed interface (Command Center, Stats, Project Marketplace, Curated Tools) |
| `/docs/$slug` | Individual documentation page with markdown rendering, anchor links, and search highlighting |

The route tree (`src/routeTree.gen.ts`) is auto-generated by the Vite plugin — commit it, never hand-edit it.

## Utility files (`src/utils/`)

| File | What it does |
|---|---|
| `helpers.ts` | Path constants (`PROJECT_ROOT`, `PLUGINS_DIR`, `RULES_DIR`, `MARKETPLACE_JSON`, `DOCS_DIR`) and `parseFrontmatter()` for YAML frontmatter in markdown |
| `plugins.ts` | `getInstalledPlugins()` (via `@repo/claude-fs`) + `getProjectMarketplace()` (reads `plugins/*/plugin.json`, `SKILL.md`, `AGENTS.md`) |
| `marketplace.ts` | `getCuratedPlugins()` — fetches from two hardcoded marketplaces (`claude-plugins-official`, `astral-sh`) via `@repo/claude-fs` cache |
| `rules.ts` | `getRules()` — reads all `.md` files in `rules/`, parses frontmatter for name/description/paths |
| `commands.ts` | `getClaudeCodeCommands()` — parses a markdown table in `docs/claude-code.md` to extract CLI commands |
| `docs.ts` | `listDocs()`, `getDocContent()`, `getAllDocsForSearch()` — slug-validated doc loading with per-heading search sections |

## Docker setup

```bash
cd apps/help-server
docker compose up --build
# App available at http://localhost:3008
```

The container mounts `../..` (repo root) at `/app`. `node_modules` and the dist output are excluded via anonymous volumes — if dependencies change, run `docker compose down -v && docker compose up --build`.

Key volume mounts from `docker-compose.yml`:

| Host path | Container path | Access |
|---|---|---|
| `../..` | `/app` | rw |
| `../../docs` | `/docs` | read-only |
| `../../plugins` | `/plugins` | read-only |
| `../../rules` | `/rules` | read-only |
| `../../.claude-plugin` | `/.claude-plugin` | read-only |
| `~/.claude` | `/root/.claude` | read-only |

## Adding a new tab / data source

1. Add a server function in a new (or existing) `src/utils/*.ts` file using `createServerFn`.
2. Call it in the `src/routes/index.tsx` loader alongside the existing parallel calls.
3. Pass the data down to a new component under `src/components/tabs/`.
4. Add a `<TabsTrigger>` / `<TabsContent>` pair in the home route component.

## Adding a new doc page

Drop a `.md` file into `docs/`. The slug is derived from the filename. `listDocs()` and `getAllDocsForSearch()` pick it up automatically — no registration needed.

## Known gotchas

- Tailwind v4: `--tailwind` flag is deprecated; all config is in `styles.css`.
- `src/routeTree.gen.ts` is auto-generated — never hand-edit.
- shadcn components must be added per-component: `npx shadcn@latest add <component>` → lands in `src/components/ui/`.
- `docs.ts` validates slugs against a regex to prevent path traversal — keep that check if you modify `getDocContent()`.
